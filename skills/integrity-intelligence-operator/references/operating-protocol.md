# Operating protocol

## Before implementation

1. Read the relevant memory files.
2. Read the relevant comparative notes.
3. Check whether the new work changes architecture, source coverage, or risk logic.

## During implementation

1. Keep raw-source provenance.
2. Separate deterministic facts from derived signals.
3. Prefer explainable, modular rules over complex scoring shortcuts.
4. Keep Paraguay-first choices portable to other countries when practical.

## After implementation

1. Update `memory/run-log.md`.
2. Update `memory/decisions.md` if a real decision was made.
3. Update `memory/source-status.md` if source work changed.
4. Update research files if a reference materially influenced the work.
5. Leave clear next actions in `memory/next-steps.md`.
